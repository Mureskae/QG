# QG Protocol Layer

This directory defines the conceptual protocol layer of QG.
It describes how attention, presence, and resonance interact as system primitives.

## Core idea

QG protocol, as described in this directory, is not executable on its own.
It defines relational rules between states of awareness and interaction.

## Components

### 1. Presence Layer

Defines minimal state of existence in the system.

### 2. Attention Layer

Defines directional focus and energy allocation.

### 3. Resonance Layer (PoR)

Defines mutual alignment between entities.

## Function of this layer

- interpret interaction
- define alignment conditions
- describe emergence patterns

## Note

This directory is a conceptual structure, not implementation.

---

*Scope note: the executable counterpart to these three layers is
`QG_Model_Spec.md` §3 (Presence, Resonance, Coherence as computed
functions) and its reference implementation, `qg_model.py` /
`qgmath/`. That code does compute concrete numbers and token-mint
decisions from these primitives — this directory describes the
relational rules in prose; the spec and code make them numeric and
testable. See also `core/` in this repository for another
implementation attempt, which should be reconciled with
`qg_model.py`/`qgmath/` rather than treated as a second, independent
source of truth.*
