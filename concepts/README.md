# QG Concepts Layer

This layer defines the core vocabulary of the QG system.
These are not definitions in a linguistic sense. They are operational primitives of a conceptual protocol.

## 1. Presence

The minimal state of being in the system.
Presence does not act. It enables interaction to occur.

## 2. Attention

Directional allocation of awareness.
Attention defines what becomes structurally relevant.

## 3. Resonance (PoR)

A condition of mutual alignment between two or more entities.
Resonance is not a signal. It is a relational state.

## 4. Alignment

Compatibility between systems, intents, or states.
Alignment is the precondition for resonance.

## 5. Emergence

Unexpected structure arising from aligned interaction.
Emergence is not designed. It happens.

## 6. Protocol

A set of rules describing interaction between conceptual primitives.
In QG, protocol replaces application logic.

## 7. System Principle

QG does not compute outcomes.
It describes conditions under which outcomes become inevitable.

## End

These concepts are not theoretical.
They are structural.

---

*Scope note: this is the philosophy layer, describing intent and
relationships between concepts in prose, not implementation. The
computed side of Presence, Resonance, and Coherence — actual numbers
from actual formulas — is defined and tested in `QG_Model_Spec.md` and
`qg_model.py` / `qgmath/`. Where this document says "QG does not compute
outcomes," it is describing the design philosophy behind the primitives
above, not the reference implementation, which does compute specific
numeric outcomes (presence scores, resonance values, token-mint
decisions). Read the two together, not as contradicting each other.*
